# 下位机视频栈配置文档

本文说明视频栈相关配置文件、launch 参数和部署环境变量。`rtsp.public_host`、摄像头设备和编码参数以 YAML 为主；systemd 环境变量只用于部署路径、launch 参数和临时覆盖。

## 1. 配置文件总览

摄像头和 RTSP 配置：

```bash
/home/rera/hostcomputer/ros1_bridge/video_sources.local.yaml
```

systemd 环境变量模板：

```bash
/home/rera/hostcomputer/deploy/systemd/robot-video-stack.env.example
```

正式 systemd 环境文件：

```bash
/etc/default/robot-video-stack
```

ROS launch 入口：

```bash
ros1_bridge/launch/video_stack.launch
```

## 2. RTSP 配置

示例：

```yaml
rtsp:
  public_host: 192.168.1.83
  publish_host: 127.0.0.1
  port: 8554
  server: mediamtx
  transport: tcp
```

配置优先级：

1. 常规部署优先修改 `video_sources.local.yaml`。
2. `RTSP_PUBLIC_HOST` 和 `RTSP_PUBLISH_HOST` 只作为临时覆盖，会覆盖 YAML 中的 `rtsp.public_host` / `rtsp.publish_host`。
3. 使用 `video_stack.sh` 时，`RTSP_PORT` 默认是 `8554`，并会传给 video manager。若要改 RTSP 端口，应同时修改 YAML 的 `rtsp.port`、systemd/环境中的 `RTSP_PORT` 和 MediaMTX 监听端口。

字段说明：

| 字段 | 必填 | 默认值 | 说明 |
|------|------|--------|------|
| `public_host` | 推荐 | `host` 或 `127.0.0.1` | 上位机看到的 RTSP 主机地址 |
| `publish_host` | 否 | `127.0.0.1` | FFmpeg 发布到 MediaMTX 的主机地址 |
| `port` | 否 | `8554` | MediaMTX RTSP 端口 |
| `server` | 否 | `mediamtx` | RTSP server 类型，当前按 MediaMTX 部署 |
| `transport` | 否 | `tcp` | FFmpeg RTSP transport，支持 `tcp` 或 `udp` |

部署建议：

```yaml
public_host: 192.168.1.83
publish_host: 127.0.0.1
```

不要把 `public_host` 配成 `127.0.0.1`，否则上位机收到的 RTSP URL 会指向上位机自己。

## 3. 摄像头配置

示例：

```yaml
direct_sources:
  - camera_id: 0
    source_id: paw_realsense
    slot_hint: 0
    name: Paw RealSense
    runner: ffmpeg
    device: /dev/paw_realsense
    input_format: yuyv422
    input_width: 1280
    input_height: 720
    width: 960
    height: 720
    fps: 30
    codec: h264
    bitrate_kbps: 2500
    vf: crop=960:720:160:0
    rtsp_path: paw_realsense
    enabled: true
```

字段说明：

| 字段 | 必填 | 说明 |
|------|------|------|
| `camera_id` | 是 | 上位机当前只支持 `0..4` |
| `source_id` | 是 | 稳定源 ID，用于识别摄像头 |
| `slot_hint` | 否 | 建议显示槽位，当前通常等于 `camera_id` |
| `name` | 否 | 人类可读名称 |
| `runner` | 否 | 当前只支持 `ffmpeg` |
| `device` | 是 | V4L2 设备路径，例如 `/dev/frontcam` |
| `input_format` | 否 | 输入格式，例如 `mjpeg`、`yuyv422` |
| `input_width` | 否 | 摄像头采集宽度 |
| `input_height` | 否 | 摄像头采集高度 |
| `width` | 否 | 对外上报输出宽度 |
| `height` | 否 | 对外上报输出高度 |
| `fps` | 否 | 帧率 |
| `codec` | 否 | 当前主要使用 `h264` |
| `bitrate_kbps` | 否 | 视频码率，单位 kbps |
| `vf` | 否 | FFmpeg video filter，当前用于裁切 |
| `crop_aspect` | 否 | 自动裁切比例，例如 `4:3`；设置 `vf` 时优先使用 `vf` |
| `video_filter` | 否 | 追加 FFmpeg video filter |
| `output_width` / `output_height` | 否 | 显式输出尺寸；不填时使用 `width` / `height` |
| `rtsp_path` | 是 | RTSP path，必须是单段路径 |
| `enabled` | 否 | 是否进入 video manager 运行时列表 |
| `ffmpeg_path` | 否 | FFmpeg 可执行文件路径，默认 `ffmpeg` |
| `gop` | 否 | H.264 GOP；不填时默认等于 `fps` |
| `keyint_min` | 否 | H.264 最小关键帧间隔；不填时默认等于 `gop` 或 `fps` |
| `x264_params` | 否 | x264 参数，默认 `repeat-headers=1` |
| `extra_input_args` | 否 | 追加到输入前的 FFmpeg 参数列表 |
| `extra_output_args` | 否 | 追加到输出前的 FFmpeg 参数列表 |

当前统一裁切策略：

```yaml
input_width: 1280
input_height: 720
width: 960
height: 720
vf: crop=960:720:160:0
```

含义是从 `1280x720` 输入中心裁切出 `960x720`，输出 4:3 画面。

## 4. 当前默认摄像头

当前启用 5 路：

| camera_id | source_id | device | rtsp_path |
|-----------|-----------|--------|-----------|
| `0` | `paw_realsense` | `/dev/paw_realsense` | `paw_realsense` |
| `1` | `overheadcam` | `/dev/overheadcam` | `overheadcam` |
| `2` | `pawcam` | `/dev/pawcam` | `pawcam` |
| `3` | `frontcam` | `/dev/frontcam` | `frontcam` |
| `4` | `armcam` | `/dev/armcam` | `armcam` |

`backcam` 当前保留但禁用：

```yaml
- camera_id: 5
  source_id: backcam
  enabled: false
```

注意：`enabled: false` 的源不会进入运行时列表，也不会上报给上位机。

当前实现会跳过 `enabled: false` 的源后再校验运行时列表，所以可以保留 `camera_id: 5` 的 `backcam` 作为未来配置；一旦启用，需要先确认上位机和协议已支持该 ID。

运行时校验规则：

- 启用的 `camera_id` 必须在 `0..4`，且不能重复。
- 启用的 `rtsp_path` 不能重复，并且只能是单段路径，例如 `frontcam`。
- `rtsp.transport` 只能是 `tcp` 或 `udp`。
- `runner` 当前只支持 `ffmpeg`。
- `codec: copy` / `codec: passthrough` 不能和 `vf`、`crop_aspect`、`video_filter` 同时使用。

## 5. FFmpeg 命令生成

`video_manager_node.py` 会按配置生成类似命令：

```bash
ffmpeg \
  -hide_banner -loglevel warning \
  -f v4l2 \
  -input_format mjpeg \
  -video_size 1280x720 \
  -framerate 30 \
  -i /dev/frontcam \
  -an \
  -vf crop=960:720:160:0 \
  -c:v libx264 \
  -pix_fmt yuv420p \
  -profile:v baseline \
  -preset ultrafast \
  -tune zerolatency \
  -g 30 \
  -keyint_min 30 \
  -sc_threshold 0 \
  -x264-params repeat-headers=1 \
  -b:v 2500k \
  -rtsp_transport tcp \
  -f rtsp rtsp://127.0.0.1:8554/frontcam
```

关键低延迟参数：

| 参数 | 作用 |
|------|------|
| `-preset ultrafast` | 降低编码延迟 |
| `-tune zerolatency` | x264 零延迟模式 |
| `-g 30` | 30 fps 下约 1 秒一个关键帧 |
| `-keyint_min 30` | 固定关键帧间隔 |
| `-sc_threshold 0` | 禁用场景切换触发关键帧 |
| `repeat-headers=1` | 新客户端更快拿到 SPS/PPS |

## 6. launch 参数

正式启动入口：

```bash
roslaunch ros1_bridge video_stack.launch
```

常用参数：

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `video_config` | `$(find ros1_bridge)/video_sources.local.yaml` | 视频源配置路径 |
| `log_dir` | `/tmp/rtsp_logs` | FFmpeg 日志目录 |
| `video_manager_host` | `127.0.0.1` | video manager 控制口监听地址 |
| `video_manager_port` | `18081` | video manager 控制口端口 |
| `rtsp_port` | `8554` | MediaMTX RTSP 端口 |
| `start_mediamtx` | `false` | 是否由 launch 启动 MediaMTX |
| `mediamtx_bin` | `mediamtx` | MediaMTX 可执行文件 |
| `mediamtx_extra_args` | 空 | 透传给 MediaMTX launcher 的额外参数，例如 `--config /etc/mediamtx/mediamtx.yml` |
| `bridge_mode` | `--ros` | bridge 模式，可设为 `--dry-run` |
| `bridge_host` | `0.0.0.0` | bridge 监听地址 |
| `bridge_port` | `9090` | 上位机 TCP/JSON 端口 |
| `debug_host` | `0.0.0.0` | debug UI 地址 |
| `debug_port` | `18080` | debug UI 端口 |
| `video_manager_timeout` | `5.0` | bridge 访问 video manager 的超时 |
| `video_manager_extra_args` | 空 | 透传给 `video_manager_node.py` 的额外参数 |
| `bridge_extra_args` | 空 | 透传给 `host_bridge_node.py` 的额外参数 |
| `respawn` | `true` | 节点退出后是否由 roslaunch 拉起 |
| `respawn_delay` | `2.0` | respawn 延迟秒数 |

示例：

```bash
roslaunch ros1_bridge video_stack.launch
```

生产部署推荐系统服务管理 MediaMTX，因此默认不由 launch 启动。若只是本机快速回环测试：

```bash
roslaunch ros1_bridge video_stack.launch start_mediamtx:=true
```

临时 dry-run：

```bash
roslaunch ros1_bridge video_stack.launch bridge_mode:=--dry-run video_manager_extra_args:=--dry-run
```

临时覆盖 RTSP public host：

```bash
roslaunch ros1_bridge video_stack.launch video_manager_extra_args:="--rtsp-public-host 192.168.1.99"
```

## 7. `video_stack.sh` 环境变量

`video_stack.sh` 现在主要用于 `status` 和兜底排障。它仍可通过环境变量覆盖部署行为。

| 变量 | 默认值 | 说明 |
|------|--------|------|
| `VIDEO_SOURCES_CONFIG` | `ros1_bridge/video_sources.local.yaml` | 视频源配置路径 |
| `PID_DIR` | `/tmp/robot_video_stack` | PID 文件目录 |
| `LOG_DIR` | `/tmp/rtsp_logs` | 节点和 FFmpeg 日志目录 |
| `VIDEO_MANAGER_HOST` | `127.0.0.1` | video manager 控制口监听地址 |
| `VIDEO_MANAGER_PORT` | `18081` | video manager 控制口端口 |
| `BRIDGE_HOST` | `0.0.0.0` | bridge 监听地址 |
| `BRIDGE_PORT` | `9090` | 上位机 TCP/JSON 端口 |
| `DEBUG_HOST` | `0.0.0.0` | debug UI 地址 |
| `DEBUG_PORT` | `18080` | debug UI 端口 |
| `RTSP_PORT` | `8554` | MediaMTX RTSP 端口 |
| `BRIDGE_MODE` | `--ros` | bridge 模式，可设为 `--dry-run` |
| `ROS_SETUP` | 空 | ROS workspace setup 脚本 |
| `VIDEO_MANAGER_DRY_RUN` | `0` | 设为 `1` 时 video manager 不启动真实 FFmpeg |
| `RTSP_PUBLIC_HOST` | 空 | 临时覆盖 YAML 中的 `rtsp.public_host` |
| `RTSP_PUBLISH_HOST` | 空 | 临时覆盖 YAML 中的 `rtsp.publish_host` |
| `VIDEO_MANAGER_TIMEOUT` | `5.0` | bridge 访问 video manager 的超时 |
| `VIDEO_MANAGER_WAIT_SEC` | `8` | 启动时等待 video manager 端口时间 |
| `BRIDGE_WAIT_SEC` | `8` | 启动时等待 bridge 端口时间 |
| `MEDIAMTX_WAIT_SEC` | `8` | 启动时等待 MediaMTX 端口时间 |
| `SKIP_MEDIAMTX_START` | `0` | 设为 `1` 时不尝试 `systemctl start mediamtx` |
| `SKIP_ROS_MASTER_CHECK` | `0` | 设为 `1` 时跳过 ROS master 检查 |
| `STOP_MEDIAMTX` | `0` | `stop` 时是否同时停止 MediaMTX |

示例：

```bash
./video_stack.sh status
```

脚本兜底启动 dry-run：

```bash
BRIDGE_MODE=--dry-run VIDEO_MANAGER_DRY_RUN=1 ./video_stack.sh start
```

临时覆盖 RTSP public host：

```bash
RTSP_PUBLIC_HOST=192.168.1.99 ./video_stack.sh restart
```

## 8. systemd 环境文件

模板：

```bash
deploy/systemd/robot-video-stack.env.example
```

复制到：

```bash
/etc/default/robot-video-stack
```

推荐内容：

```bash
HOSTCOMPUTER_DIR=/home/rera/hostcomputer
VIDEO_SOURCES_CONFIG=/home/rera/hostcomputer/ros1_bridge/video_sources.local.yaml
RTSP_PORT=8554
START_MEDIAMTX=false
MEDIAMTX_BIN=mediamtx

VIDEO_MANAGER_HOST=127.0.0.1
VIDEO_MANAGER_PORT=18081
BRIDGE_HOST=0.0.0.0
BRIDGE_PORT=9090
DEBUG_HOST=0.0.0.0
DEBUG_PORT=18080

LOG_DIR=/tmp/rtsp_logs
PID_DIR=/tmp/robot_video_stack

ROS_SETUP=/home/rera/robot24_ws/devel/setup.bash
```

systemd 模板会 source `ROS_SETUP`，把 `HOSTCOMPUTER_DIR` 加入 `ROS_PACKAGE_PATH`，然后运行：

```bash
roslaunch ros1_bridge video_stack.launch
```

MediaMTX 服务模板在：

```bash
deploy/systemd/mediamtx.service
```

`RTSP_PUBLIC_HOST` 默认不写在 systemd 环境文件里，优先在 `video_sources.local.yaml` 的 `rtsp.public_host` 中配置。临时覆盖时可以写 `RTSP_PUBLIC_HOST=192.168.1.99`，systemd 模板会转换成 `video_manager_node.py --rtsp-public-host ...`。

## 9. 修改配置后的生效方式

修改 `video_sources.local.yaml` 后重启视频栈：

```bash
sudo systemctl restart robot-video-stack
```

只想临时禁用某一路：

```yaml
enabled: false
```

然后重启：

```bash
sudo systemctl restart robot-video-stack
```
