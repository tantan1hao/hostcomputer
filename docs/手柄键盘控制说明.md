# 手柄与键盘控制说明文档

## 概述

上位机v2 支持 **键盘** 和 **手柄（SDL3 Gamepad，Windows 下保留 XInput fallback）** 两种输入输入源。上位机只采集输入快照并转发给下位机，不在本地解析车体/机械臂运动语意。

- 键盘输入以 `keyboard.pressed_keys` 发送。
- 手柄输入以 `gamepad.buttons` 和 `gamepad.axes` 发送。
- 当前上位机只保留急停语意：键盘 `Space`，或手柄 `L3+R3` 同时按下。
- 运动模式、速度档位、D-Pad、摇杆、扳机等语意由下位机 bridge/watchdog/控制节点解析。

协议仍保留 `mode` 字段以兼容现有下位机协议，但普通输入按键不再在上位机内触发模式切换。

| 模式 | 说明 |
|------|------|
| **车体模式** | 控制底盘运动（前后/转向） |
| **机械臂模式** | 控制机械臂末端位姿（笛卡尔空间） |

---

## 一、键盘输入

> 类：`KeyboardController`，发送频率 10Hz

### 1.1 转发按键

| 按键 | 功能 | 说明 |
|------|------|------|
| 字母键 | 转发按键名 | 例如 `w`、`a`、`s`、`d`、`q`、`e` |
| 数字键 | 转发按键名 | 例如 `1`、`6`、`7`、`8`、`9`、`0` |
| 方向键 | 转发按键名 | `up`、`down`、`left`、`right` |
| 修饰键 | 转发按键名 | `shift`、`ctrl`、`alt`、`tab` |
| `Space` | **急停** | 上位机直接发送 `emergency_stop` |

除 `Space` 急停外，上位机不解析按键语意。

---

## 二、手柄输入

> 类：`HandleKey`（SDL3 Gamepad API，Windows 下可回退到 XInput），轮询频率 50Hz

手柄按钮、D-Pad、摇杆和扳机均作为原始快照转发。上位机不再解析 D-Pad 模式切换，也不再把 `A` 键解释为急停。

| 输入 | 上位机行为 |
|------|------------|
| A/B/X/Y、LB/RB、Start/Back、D-Pad | 转发按钮快照 |
| L3/R3 | 转发按钮快照；同时按下时触发急停 |
| 左右摇杆 | 转发归一化轴值 `[-1.0, 1.0]` |
| LT/RT | 转发归一化轴值 `[0.0, 1.0]` |

---

## 三、急停机制

急停是最高优先级操作，**不受控制模式和启用状态影响**：

| 触发方式 | 说明 |
|---------|------|
| 键盘 `Space` | 全局快捷键，任何时候可用 |
| 手柄 `L3+R3` | 两个摇杆同时按下 |
| UI「急停」按钮 | 界面按钮 |

急停触发后执行以下操作：
1. 通过 TCP 发送 `emergency_stop` 指令到下位机
2. 清空本地输入快照
3. 重置键盘控制器状态

## 四、数据流

```
┌──────────────────────┐
│  键盘按键 / 手柄摇杆  │   ← 用户输入
└──────────┬───────────┘
           │
           ▼
┌──────────────────────┐
│  KeyboardController  │   ← 10Hz 定时发送
│  HandleKey (SDL3)    │   ← 50Hz 轮询，变化时触发
└──────────┬───────────┘
           │  Qt 信号
           ▼
┌──────────────────────┐
│     MainWindow       │   ← 汇总输入快照
│  - Space/L3+R3 急停  │
│  - 其余输入仅转发    │
└──────────┬───────────┘
           │
           ▼
┌──────────────────────┐
│     Controller       │   ← 业务逻辑层
└──────────┬───────────┘
           │
           ▼
┌──────────────────────┐
│   ROS1TcpClient      │   ← TCP JSON 发送
│   Port 9090          │
└──────────┬───────────┘
           │
           ▼
┌──────────────────────┐
│  Intel NUC (下位机)   │   ← ROS1 执行控制
└──────────────────────┘
```

## 五、UI 显示

### 手柄状态面板 (`GamepadDisplayWidget`)

- **左右摇杆圆盘**：实时显示摇杆位置（圆点跟随移动）
- **LT/RT 扳机进度条**：竖直进度条显示扳机按压深度
- **按钮状态标签**：显示当前按下的按钮名称
- **连接状态**：绿色「已连接」/ 红色「未连接」

### 状态栏信息

- 控制模式显示：来自下位机状态或协议兼容字段
- 手柄连接状态
- 命令日志（带 500ms 节流防刷屏）
