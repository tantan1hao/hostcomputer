# 上位机部署说明

## 目标

上位机发布包需要在 Windows 和 Linux 上尽量开箱即用。RTSP 低延迟播放使用 FFmpeg 子进程，但工程本身不链接 `libav*`，因此构建阶段不需要安装 FFmpeg 开发库。

协作者开发机安装 FFmpeg 的步骤见 [FFmpeg开发环境安装指南.md](FFmpeg开发环境安装指南.md)。本文件只描述正式发布包如何随带 FFmpeg。

## 运行时目录

推荐发布包目录结构：

```text
Windows:
hostcomputer/
  hostcomputer.exe
  ffmpeg.exe
  Qt6Core.dll
  Qt6Widgets.dll
  ...

Linux:
hostcomputer/
  hostcomputer
  ffmpeg
  libQt6Core.so.6
  libQt6Widgets.so.6
  ...
```

FFmpeg 查找顺序：

1. `HOSTCOMPUTER_FFMPEG_PATH` 环境变量。
2. 上位机可执行文件同目录的 `ffmpeg.exe` 或 `ffmpeg`。
3. 系统 `PATH`。

为了开箱即用，发布包应采用第 2 种方式：把 FFmpeg 可执行文件随包放在上位机可执行文件同目录。

## 发布检查

本机开发环境可以允许使用系统 `PATH` 里的 FFmpeg：

```bash
python3 scripts/check_host_runtime.py \
  --app-dir build/Desktop_Qt_6_7_3-Debug
```

正式发布包必须检查 FFmpeg 是否随包：

```bash
python3 scripts/check_host_runtime.py \
  --app-dir /path/to/release/hostcomputer \
  --require-bundled-ffmpeg
```

Windows 上同理：

```powershell
python scripts/check_host_runtime.py `
  --app-dir C:\path\to\release\hostcomputer `
  --require-bundled-ffmpeg
```

## FFmpeg 来源

仓库不直接提交 FFmpeg 二进制，避免把大体积平台产物混进源码。发布时按目标平台选择一个可再分发的 FFmpeg 构建，并把对应的许可证文件随发布包一起提供。

注意：

- 如果使用 GPL 版 FFmpeg 构建，发布包需要满足 GPL 相关义务。
- 如果希望降低许可证约束，应选择 LGPL 配置的 FFmpeg 构建，并保留对应 license notice。
- Windows 和 Linux 的 FFmpeg 文件名不同：Windows 为 `ffmpeg.exe`，Linux 为 `ffmpeg`，Linux 版需要有可执行权限。

## Qt 运行时

Windows 发布包应使用 Qt 自带的 `windeployqt` 收集 Qt 运行时 DLL 和插件。Linux 发布包可以使用系统 Qt、AppImage、linuxdeployqt，或项目后续固定的发布方式。无论采用哪种 Qt 部署方式，都需要把 `ffmpeg` 放入最终可执行文件所在目录。
